数据处理

one-hot的原因：进行向量加减运算有意义

删低频词：name entity，typo，vocab太大导致one-hot太稀疏（参数太多）

步骤：

  1. tokenization（大小写转换、stop word、拼写纠错等）
  2. 词频统计排序索引
  3. one-hot encoding（不存在的词可以编码为0）
  4. sequence alignment（trimming，zero-padding）

Simple-RNN

前时刻的hidden state + 当前时刻的input = 当前时刻的hidden state

激活函数：防止hidden state爆炸

缺点：短期记忆而非长期记忆（长期的导数趋近于0？？？）
