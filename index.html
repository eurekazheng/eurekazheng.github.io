当人们喜欢编程时，在喜欢什么？

发表于 2022-05-24 评论次数： 阅读次数：

计算机科学（包括信息科学、数据科学、人工智能、离散数学等分支）和商学是绝无仅有的两个实用性特别强的基础设施性学科。这意味着它们为社会生活当中的几乎所有任务搭设基础设施（数学对于大部分任务的实用性并不是非常高）。这也正是它们的从业人员薪资较其他行业高的原因。

在这两个学科当中，计算机科学的创造性又更强一点，属于直接解决问题本身（如通过算法直接处理分析核心数据，执行业务逻辑）。而商学还只是为特定方面服务（任务的商业市场行为，如营销、投融资、财税法务等）。

计算机科学有工程（解决特定问题的方法论）和科学（为了解决问题的技术储备）两方面。针对题主提到的工程（编程）方面，我从工程中获得的满足感除了任务得以实现的成就感之外，还有工程本身的实用主义美学（也即“代码的艺术”）：最佳实践和设计模式所具备的极简主义、可读、鲁棒、解耦（模块化、可复用）、可扩展等特性往往令人叹为观止。

我一直认为计算机工程师需要具备一定的审美修养。

第二，程序的可复制性意味着它更容易在线合作，并孕育了自由软件运动等思潮，倡导一种前所未有的获取知识和技术的便利性。

发表于 2022-05-21 更新于 2022-05-22 评论次数： 阅读次数：

JSON (JavaScript Object Notation) is a raw data representation scheme that supports hierarchical positional and keyword indexing (retrieval). It's isomorphic to YAML.

HTML (HyperText Markup Language) is a markup scheme for any (static or interactive) document.

  • Each piece of content (node) has its type, properties (styles), methods (effects), and events. They are organized in a hierarchical manner to form DOM (Document Object Model):

    HTML is similar to other rich-text languages or specifications such as XML, Office Open XML (behind Microsoft Word), PDF, TeX, and Markdown (can be cross-converted using Pandoc).

Scss (Sass (Syntatically Awesome Stylesheets), Stylus, Less, CSS) implements the DOM (basically) static styles (very limited cross-node event, stateless).

TypeScript (Node.js，JavaScript) implements the dynamic business logic (heavily cross-node, stateful) of DOM.

  • Traditional plain JavaScript (e.g. jQuery) uses a flat selector (e.g. $('.node')) - modifier (e.g. .src='new target', .hide(), .change(function() {...})) scheme to directly manipulate the DOM.

    Modern TypeScript or Node.js (e.g. React, Vue.js, AngularJS) manipulates DOM by via a medium or interface that represents states of the application (states in React or data in Vue.js). States decouple manipulation in a publisher (how to change states on DOM events) - subscriber (how to refresDOM from states) pattern (JSX in React or template in Vue.js):

    , and make it more declarative than imperative. However, they need to be built (using e.g. webpack) as plain JavaScript for production environment.

    Manipulation can be encapsulated in a hierarchical (object-oriented) manner (components) and published as UI libraries that incorporate design systems (e.g. Ant Design, Element Plus, Material UI for Node.js and Bootstrap (before 5.0) for jQuery) or domain-specific functionalities:

For non-document (highly complex or interactive) content such as 3D models, more holistic tools are applied:

  • Data Science & Engineering: D3.js
    • Data Visualization: Apache ECharts, Highcharts JS, Chart.js
  • Graphics: WebGPU, WebGL, twgl.js, Babylon.js
    • Physically-based Rendering (PBR): Filament.js
  • 3D Model: Three.js
    • Digital Twin: ThingJS
  • Animation: Anime.js
  • Procedural Art: Processing
  • GIS: ArcGIS API for JavaScript, Cesium, OpenLayers, Leaflet

For computation occurred at frontend (edge computing):

  • Deep Learning: Tensorflow.js, ONNX.js (for PyTorch etc.)
  • Native: WebAssembly (WASM) with C/C++ or Rust

关于阅读记录的一些体会

发表于 2021-05-26 更新于 2022-03-07 评论次数： 阅读次数：

非审美阅读：感知机算法

感知机（Perceptron）算法
  • 形成自己的结构，而不是作者的结构（“六经注我”，而不是“我注六经”）：将单独一条高维的原始文本映射到低维的诠释文本，可能存在最优解（通过PCA）。但是，阅读是一个在线学习（Online Learning）的过程。以感知机（Perceptron）算法举例，每个人接触到的文本分布 \(\mathcal{X}\) 与积累的文本集 \(\mathbb{x}\) 不同，对当前文本 \(\mathbb{x}_t\) 的诠释 \(\hat{y}_t\) 也不同，乃至于对诠释模型 \(\mathbb{w}\) 的更新也不同。比如我在机器学习方面的文本分布 \(\mathcal{X}\) 和我之前阅读过的介绍感知机的文本 \(\mathbb{x}\)：Pattern Recognition & Machine Learning，决定了我这种稍显特色的举例及其具体的notation
    • 大胆猜想：缺乏资料的时候，不妨先做假设，再斟酌取舍。或从其他模型借用，比如我从计算机类的模型来套用其他技术类的
    • 跳读与归纳：翻看（非审美阅读的有效比例只有1%~25%），总结出要点，一般是特别标注或居特定位置的话，有时候全章只有一两句。看《未来医疗》并写读书笔记时，用了大概4天的闲暇时间
    • 收藏：不同于归纳，有意义的例子值得收藏，毕竟例子能够印证鉴别作者的一些说法。收藏的程度是能够让日后检索
    • 选择性：挑选感兴趣，对自己的 \(\mathbb{w}\) 有意义的要点，比如颠覆性或匮乏的（\(\hat{y}_t \neq y_t\)），而不是全部要点
