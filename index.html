Stack, Queue, and List 棧、隊列與列表

發表於 2017-12-09 | 分類於 Data Structures and Algorithms 數據結構與算法

本文介紹的棧、隊列與列表，在C++ STL術語中屬於線性序列容器(Linear Sequence Containers)，更高級的序列容器有樹、圖等、與非序列的集合容器(Associative Containers)相對。

數組與鏈表的基本權衡：數組比鏈表(特別是雙向鏈表)少佔用指針域，但數組通常佔用冗餘空間，而鏈表按需佔用。此外，數組容量不夠時需要$O(n)$的擴充拷貝。

棧 Stack

棧是最基礎的數據結構，有先入先出(First In Last Out, FILO)的特點。棧的操作一般只有入棧(Push)、出棧(Peek)和訪問最頂(Top)。數組和鏈表實現效率都是$O(1)$，除非當數組容量不足擴充數組時須耗時$O(n)$進行拷貝。

列表 List

與棧或隊列不同，列表可隨機訪問其中元素(Random Access)，而不僅是頂或者首尾元素。主要功能有插入(insert)、按索引刪除(remove by index)、按值刪除(remove by value)、獲取(get)、獲取索引(indexOf)、修改(set)、判斷包含(contains)等。

插入、刪除等改動數組大小的方法，鏈表實現難度在於遍歷到達指定位置，數組實現難度在於之後元素的復位，故而改動位置在數組中的前後也影響到了鏈表與數組的權衡取捨，但就獲取、修改而言，數組就優於鏈表。

鄰接表(Adjacency List)：鏈表佔用額外空間且邊查找需遍歷終點鏈表，但按需佔用及遍歷，適用於稀疏圖。鄰接表有多種具體實現方式，包括鏈表的數組、數組的數組(注意同二維數組不一樣)、

圖的性質

無權圖的最短路算法以廣度優先搜索實現。
有向圖：二叉堆(Binary Heap)等數組實現的完全二叉樹(Array Complete Binary Tree)、雙向鏈表(Doubly Linked List)。

二叉查找樹

插入或查找：$\mathrm{lg}n$：取決於數是否足夠平衡，在最壞情況下$\mathrm{lg}n$退化爲$n$(鏈表)
刪除：$1$：只需將目標節點與最左右孩子(Leftmost right child)或最右左孩子(Rightmost Left Child)交換。注意每次交換時選擇最左或最右須交替進行，以平衡左右子樹。

集合與字典的二叉查找樹實現

二叉查找樹按照值或鍵的散列值排序集合或字典的元素，雖然效率($\mathrm{lg}n$)不比散列表($n$)，但可以避免在遍歷碰撞元素和擴充數組時重新對所有元素排址的開銷。雖然佔用更多空間(每節點兩個指針)，但完全按需。

廣度優先搜索 Breadth-first Search, BFS

廣度優先搜索的思路是從根節點入隊開始，將當前節點的孩子入隊後自己出隊，直到遇見的第一個解(葉子)即是最優解。但相較採用貪心策略(Greedy Strategy)的深度優先搜索(Depth-first Search, DFS)，卻不一定是 找到一個有效解的最快方法。由於BFS的不可撤銷性，需要一字典記錄某一節點的父節點。同時，BFS也可以用訪問記錄集合剪枝。BFS是一種無權圖(Unweighted Graph)最短路徑算法。

堆 Heap

堆是隊列的樹實現。最大堆(Max Heap)是優先隊列(Priority Queue)的二叉樹實現。它是一種二叉堆(Binary Heap)。
插入：$\frac{\mathrm{lg}n}{2}$，放到最後，再上濾滿足堆序性。
刪除堆頂：$\mathrm{lg}n$，在排序算法中有介紹。

二叉堆的實現

因爲二叉堆的完全性(Completeness)，它可通過數組實現。令父索引$p$、左子索引$l$、右子索引$r$，子索引$c$，則有$l=2p+1,r=2p+2,p=\frac{c-1}{2}$。因此，二叉查找樹及其他用鏈式結構實現的樹是單向的，而數組實現的二叉堆是雙向的。

Sorting Algorithms 排序算法

發表於 2017-11-07 | 分類於 Data Structures and Algorithms 數據結構與算法

排序算法

爲更加精確地比較各種排序算法，我們計算其準確運行次數而非其複雜度(漸進上界)。以整數作爲數據。

選擇排序 Selection Sort

選擇排序每次將最大元素置頂，循環n次。具體來說，在第i次遍歷中，從0遍歷至size-i，將最大元素與第size-i元素交換。其優勢在於無論何種情況只需交換n次(交換相當於三次賦值操作)。

比較：$\frac{n^2}{2}$ (分母2來自於不斷縮小的待排序部分) 交換：$n$

賦值(記錄最大值與其索引)：對於隨機分佈的$n$個元素，第$n$個元素大於之前所有元素的概率是$\frac{1}{n}$(最大元素在第$n$位)；所以第$i$個元素大於之前所有元素的概率是$\frac{1}{i}$($1-i$子數列最大元素在第$i$位)。所以賦值期望值$$\mathrm{E}(Assignments)=\Sigma_{i=2}^{n}\Sigma_{j=1}^{i}\frac{1}{j}=n-1+\Sigma_{i=2}^{n}\frac{n-i+1}{i}=(n+1)\Sigma_{i=2}^{n}\frac{1}{i}\approx (n+1)(\mathrm{lg}n+C)$$，其中歐拉常數$C\approx 0.5722$

空間：$O(n)$

插入排序 Insertion Sort

插入排序對每個元素往前搜索直到合適的位置，因而當前元素之前的子數組總是已排序好的(可以通過歸納法理解)。插入排序適合於實時維護數組的有序性(優先隊列)。插入排序的時間效率有顯著提高，但在數組逆序程度大時不理想。

數組實現：運行=交換(平均)：$\frac{n^2}{4}$ (分母4來自於不斷增大的可插入範圍和插入移動次數的期望) 最壞：$\frac{n^2}{2}$ (完全逆序，插入移動次數爲滿) 最優：$n$ (完全正序，插入移動次數爲0) 空間：$O(n)$

鏈表實現：將往前的不斷交換換爲比較

希爾排序 Shell Sort

希爾排序改進自插入排序，依照步長序列，每次以一步長進行插入排序。插入排序是一種步長序列爲{1}的特殊希爾排序。在好的步長序列下，希爾排序能使元素以步長序列比插入排序(線性逼近)更快地逼近指定位置，數據移動少，當數組逆序程度大時優勢更加顯著。

歸併排序 Merge Sort

歸併排序是採用分治策略，每次通過比較兩個棧頂元素，整合兩個已經排序好的棧。故而歸併排序須採用後序遞歸。歸併排序由於讀取時改動，需要創建臨時數組然後回傳。歸併排序由於不同子進程之間互不影響，分佈式計算性能良好。

比較：$n\mathrm{lg}n$ 數組回傳：$n\mathrm{lg}n$ 空間：$n+\mathrm{lg}n$

原地歸併排序 Inplace Merge Sort

快速排序 Quick Sort

快速排序也是一種分治策略。通過將元素按照與基準(Pivot)的大小關係分類到點的兩側，創造兩個更小的排序任務，故而使用先序遞歸。這裏取基準爲首、中、尾三元素的中位值。

比較(平均、最優(每次基準恰好居中))：$n\mathrm{lg}n$ 最差(每次基準恰好是最大/最小值)：$\frac{n^2}{2}$ 交換(平均)：$\frac{n\mathrm{lg}n}{4}$ 最優(每次都不用交換)：$0$ 最差(基準爲最大/最小值且每次都須交換)：$\frac{n^2}{4}$

空間：$O(n)$

基準的放置，以及爲了得到儘量居中的基準的額外計算，使得快速排序對小數組效率並不顯著。在上述代碼中，小於5個元素的數組退而使用插入排序。

堆排序 Heap Sort

堆排序本質上是選擇排序的一種優化。利用最大堆(Max Heap)棧頂元素保持爲最大元素的特點，以及刪除堆頂元素後重新調整堆以滿足堆序性(Heap order)的過程(Heapify, 下濾(Percolate down; Re-heap down))的效率($\frac{\mathrm{lg}n}{2}$, 相比較選擇排序的$n$)。而選擇排序相較而言則每次只記錄最大值，交換次數甚少。注意：這裏下濾效率的分母2是由於刪除過程中堆在不斷縮小，而與下濾次數期望無關(事實上，下濾次數期望爲$\mathrm{lg}n$，因爲待下濾的堆頂來自堆的底層。由於刪除操作涉及堆的結構，這種做法是難以避免的(空位必須傳遞$\mathrm{lg}n$次才能補全))。

堆排序和選擇排序可用於“Top K問題”。

散列函數

散列函數(Hash function)，又稱哈希函數，是從任意類型或類映射至一整數(散列值，Hash value)的函數。Java的類Object聲明瞭hashCode方法，默認基於對象地址，因此可能違背下文提到的確定性特性，故而hashCode須應需重寫。

字符串散列函數(並非理論上最緊湊的排布)：

長整型與雙精度浮點數散列函數：左右兩半取抑或(XOR)。

散列函數特性

確定性 Determinism

兩次相同輸入的散列函數應返回相同結果。如其本身並非輸入數據，函數不應基於時間(Execution time)、對象地址(Object address)、可以在不改變對象身份的前提下改變值的數據(Transient Data，如人的名字、商品的價格)等。

均勻性 Uniformity

對於滿足任意分佈的一組輸入，散列函數的輸出應均勻分佈於其值域(整數域)，使得相鄰輸入會被拉開。可以生成多組不同隨機分佈的輸入，對每組的輸出進行卡方檢驗(Chi square test)。本博客採用JUnit進行性能測驗。testUniformity中的refreshRange、renew等Hashable接口方法將會在下文實現。

抗碰撞性 Collision Resistance

當散列函數不是雙射時，會存在兩個不同的輸入被映射到同一散列值，稱爲碰撞(Collision)。這裏從效率角度來講，散列函數應該儘量避免碰撞。但事實上，抗碰撞性是個比均勻性更弱的需求。更均勻的輸出分佈意味着更少的碰撞。以下給出一個關聯性分析。

加密散列函數 Crytographic Hash Function

加密散列函數滿足基本的三條特性：

  1. 抗原象攻擊(Preimage Resistance)：對散列函數$h:X\rightarrow Y$，計算$h^{-1}$是困難的。
  2. 抗第二原象攻擊(Second Preimage Resistance)：已知$x \in X$，求$x’ \in X$，滿足$h(x’)=h(x)$是困難的。又稱弱抗碰撞性。
  3. 抗碰撞性：這裏從抗碰撞攻擊角度將，定義爲求$x, x’ \in X$, 滿足$h(x)=h(x’)$是困難的。又稱強抗碰撞性。

但仍然有諸多產生碰撞的方法，提供時間戳造假和散列碰撞拒絕服務攻擊(Hash Collision DoS Attack)。

時間戳(Timestamp)，即通過在某時間公佈某文檔的散列值，證明該文檔在該時間之前已經存在。由加密散列函數的抗原象攻擊性，很難在事後產生特定散列值的文檔。而且因爲文檔通常爲證書、知識產權等有意義內容，事後生成更加困難。

拉鍊法 Seperate Chaining

採用拉鍊法解決碰撞時，容器數組不直接存放元素，而是碰撞元素的鏈表。常用於實現字典(Dictionary)。以下就分別以線性探測和拉鍊法實現集合與字典。

集合 Set

作爲最簡單的關聯(無序)容器(Associative container)，一個集合通常只實現判斷包含(contains)的訪問方法(Accessor)。而採用散列表實現可加速其搜索過程(不需要遍歷容器數組，只需基於散列值獲得存儲地址)。

字典 Dictionary

字典的核心訪問方法是搜索(通過鍵尋找值)。注意我們使用拉鍊法解決碰撞，在插入、計算地址時從整數域縮放到”[0, 數組長度)”，額外引入容器數組長度，會令碰撞的元素可能具有不同的鍵散列值(碰撞元素的鍵散列值可以是與其地址關於數組長度同餘的所有整數)，插入元素時根據鍵散列值選擇恰當位置，可進一步加速搜索過程。

可以觀察到，集合其實同構於的特殊字典。

集合與字典有時也用二叉查找樹(Binary Search Tree)實現(稱爲Tree Set或Tree Dictionary)。後者雖然正常添加、查找、刪除的效率($O(\mathrm{lg}n)$)不如散列表($O(n)$)，且相較兩種衝突解決方案都佔用額外空間；但卻避免了衝突解決帶來的如添加及查找時線性探測、遍歷衝突鏈表、刪除及增表時重新計算地址等的一系列開銷。當使用者對數據的量和分佈情況無法確定的時候，可以使用BST避免衝突解決風險。(注意：由於二叉查找樹不一定是完全二叉樹(Complete Binary Tree)，其如果用數組實現將會十分冗餘，故不予考慮)。
